**场景再现：** 主包今天遇到了写`BLOG`的第一个大问题——客户端与服务端。起因是主包在`page`界面中加入了一个点击响应的按钮，所以在`page`界面中加入了“`use client`”语句；主包还有一个工具函数库`post.tsx`，在这里主包用到了`fs`函数。此时主包想要在`page`中调用`post.tsx`中的一个用到`fs`的函数，出现经典问题了——`Module not found: Can't resolve 'fs'`

**错误原因：** 因为`fd`函数只能运行在服务器端，而我的`page`界面引用`use client`后变成了客户端！在这里主包犯了新手很容易犯的两个错误。

1. 我们约定俗成`page`界面应该是服务端的，如果在`page`中想用响应式按钮，应该新开一个文件作为客户端，在这个新开的文件中写响应式按钮，再回到`page`中`import`此按钮。
2. 一个好的、规范的前端界面，应该做好文件夹的规范命名和函数、组件的合理存放。功能函数就应该新开一个`lib`文件夹；功能组件就应该新开一个`compomnents`文件夹；最终的`page`界面应该是很简介的，大部分都是调用的，而不是像主包这样有一大坨。

**知识点补充：**

- 服务端：是不允许用户直接访问的，只能通过API/网络请求访问。在 Next.js 中指的是运行 Node.js 的服务器，它能访问文件系统 (`fs`)、数据库和环境变量（如 API Keys）。
- 客户端：是渲染UI，允许客户点击、拖拽、输入的。在 Next.js 中指的是运行在用户浏览器里的 JavaScript 代码。
  我们默认所有组件都是服务端组件，只有当你需要用户相互式操作时，才会用到客户端组件。**（记得还要规范区分文件夹！）**
